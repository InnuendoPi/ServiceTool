# ServiceTool Tasks

Dieses Verzeichnis enthält die gemeinsam gepflegten Entwicklungsaufgaben für
das Brautomat32 ServiceTool.

## Struktur

- `ACTIVE.md`: offene und laufende Aufgaben
- `archive/`: abgeschlossene oder verworfene Aufgaben

## Regeln

- Jede Aufgabe beschreibt Ziel, betroffene Dateien, Akzeptanzkriterien und
  offene Risiken.
- Laufende Arbeit wird in `ACTIVE.md` kurz nachgeführt.
- Nach Abschluss wird eine Aufgabe mit Ergebnis und Prüfstand in das Archiv
  verschoben.
- Zugangsdaten, lokale Pfade, Geräte-Backups und Testgerätedaten gehören nicht
  in Aufgaben-Dateien.

## Abgrenzung

Der Firmware-Test-Runner und `tasks/test-automation` bleiben im privaten
Repository `privBrautomat32`. Sie sind kein Bestandteil dieses Repositorys.
